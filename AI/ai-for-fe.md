# AI对前端的冲击

就算没有AI，现在对前端的要求也和以前大不相同，结合10+年前端工作经验，总结下近半年来对现有前端岗位的思考：

## 代码能力

约8年前：

- 会写Vue
- 能写jquery
- 能改样式
- 兼容性处理
- 能写基本的React代码

现在：

- Vue+TS/React+TS 撸中后台
- 解决小程序、uniapp多端编译问题，如果你还没实战经验...
- 技术栈选型，搭个工程出来，输出组件加载性能（别说你会从github、gitee上clone一个工程出来）
- 不同技术栈的快速切换（或者叫上手）能力
- 非JavaScript的技术能力，java已落伍，Go势头正盛，Rust正在改变基础能力
- 开发插件能力（多构建工具的支持），现代前端的必备技能
- 构建Electron/Tauri桌面应用
- 使用AI工具快速撸代码的能力，且保证能**正常**运行（只是正常运行）
- 好好说话的能力

这能是同一个“前端”吗？

## 被掩盖的基础能力

原来的核心要求：

- 原生Ajax封装
- 事件委托、事件冒泡、事件捕捉
- 原型链继承的方法
- Diff算法
- Promise polyfill处理

现在的情况：

- ajax 在axios中全部梭哈了
- click事件在Vue做成了指令，在React处理成了合成事件
- 继承由 `子类 extends 父类` 一把梭哈
- Diff 由框架原生支持
- Promise浏览器原生支持

这些核心要求你是否还有？你自认为对框架很熟悉了？觉得自己牛叉了？那是纯纯的幻觉，那是”必须掌握的下限”被工具抬高了。

当框架出bug、当性能要优化、当面试被问”为什么”的时候，你是不是那个”只会用、不懂原理”的人？是不是露馅了？

## 对工程化的理解

命令`npm install ` 和`npm run start`是不是敲了上百遍？后面的原理有没有想过了解一下？
这可能不是你的错，因为工具链把你从1层带到了10层，可能默认不会给你安排下楼的梯子。

## AI能干的事儿

- 生成前端工程，配置还挺全
- 自动生成页面，效果还很好
- 写业务逻辑稍弱，但是不是致命问题
- 不用关注背后的原理、技术无关，照样出效果
- 需求清楚，产品经理就可能适应前端岗

AI留给前端的自留地：

- 技术选型，这需要专业性，尤其是优化方面
- AI输出工程的纠错
- 工程化的介入
- 复杂样式的调整

## 最后的总结

前端的”动手能力”在变低，”系统能力”在变高，”中间层”在被掏空，这是不争的事实。
具体扩展成以下几个方面：

- 前端头部（架构师、性能优化专家、底层工具开发者）：能力比8年前强10倍。这部分需要懂浏览器原理、懂V8、懂Go/Rust，他们做的是基础工具Vite、Rspack、bun这样的。
- 公司前端头部（2%的人），懂得解决AI生成工程的问题、编译、打包问题，有良好的插件能力，结合公司业务能给出合适的升级方案、整合方案。这要求是全方位的能力。
- 底部的60%，能力和十年前差不多甚至更弱。但他们的产出，因为前端头部、公司头部、工具、搜索、AI加持。
- 尴尬的中间层，手写能力被新人用AI追上，架构能力又够不着真正的高手。这部分人正在被时代挤压。

在AI的时代做一个不容易被AI替代的人，难呐！
